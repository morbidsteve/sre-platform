## Deployment Paths

### Quickstart (Recommended for first deployment)

```bash
./scripts/quickstart-proxmox.sh
```

Handles everything: Proxmox discovery, cloud image import, VM provisioning, OS hardening, RKE2 install, and Flux bootstrap. See [Getting Started: Proxmox](../docs/getting-started-proxmox.md).

### Manual Deployment

1. **Provision VMs** вЂ” `cd infrastructure/tofu/environments/proxmox-lab && tofu apply`
2. **Harden + Install** вЂ” `cd infrastructure/ansible && ansible-playbook playbooks/site.yml -i inventory/proxmox-lab/hosts.yml`
3. **Bootstrap Flux** вЂ” `./scripts/bootstrap.sh`

### Packer (Production STIG images)

For full DISA STIG compliance with OpenSCAP validation:

```bash
cd infrastructure/packer/rocky-linux-9-proxmox
packer build .
```

## Key Design Decisions

- **OpenTofu over Terraform** вЂ” MPL 2.0 licensed, no BSL concerns ([ADR-008](../docs/decisions.md))
- **Rocky Linux 9** вЂ” RHEL-compatible, free, same DISA STIG applies ([ADR-009](../docs/decisions.md))
- **RKE2** вЂ” Only DISA STIG-certified K8s distribution with FIPS support ([ADR-001](../docs/decisions.md))
- **Local state backend** for homelab; S3 + DynamoDB for shared environments

## NIST 800-53 Controls

| Control | Implementation |
|---------|---------------|
| CM-2 (Baseline Config) | Immutable templates, GitOps reconciliation |
| CM-6 (Config Settings) | DISA STIG via Ansible, CIS benchmark profile |
| AC-17 (Remote Access) | SSH key-only auth, no password login |
| SC-13 (Crypto Protection) | FIPS 140-2 mode in Packer build |
| SI-7 (Integrity) | AIDE file integrity monitoring |
| AU-2 (Audit Events) | auditd rules, kernel audit logging |
